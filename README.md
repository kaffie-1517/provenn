# ProveNN (Proof of Neural Network)

ProveNN is an enterprise B2B SaaS platform that mathematically guarantees invoice integrity to prevent vendor fraud. It ensures that an invoice generated by a billing system is exactly the same one submitted by an employee for reimbursement, using cryptographic hashing and zero-trust architecture.

## Overview

The system consists of a Golang backend, a Next.js frontend, and a PostgreSQL database. It features multi-tenancy, background job processing (via River), robust observability (Prometheus & Grafana), and object storage (MinIO).

### Key Features
- **Tamper-Proof Invoicing**: Vendor invoices are stamped with a cryptographic hash and a unique 8-character reference code (e.g., `PNN-X8K9M2L4`).
- **Fraud Detection**: When an employee uploads an invoice for verification, the system hashes the PDF and compares it against the original hash. Any alteration results in a `mismatch`.
- **Multi-Tenancy**: Data is strictly segregated by `company_id`. Company admins can only see and approve verifications for their own employees.
- **Platform Admin Console (Portal 2)**: A cross-tenant dashboard for system administrators to monitor overall usage and billing events.
- **Automated Export**: Company admins can export approved verifications to a `.xlsx` spreadsheet for their ERP system.
- **Observability**: Built-in Prometheus instrumentation and Grafana dashboards for monitoring API and worker performance.

## Prerequisites

- Docker and Docker Compose
- Go 1.24+ (for local development)
- Node.js 18+ (for frontend development)
- Python 3 (for running the demo script)

## Getting Started

The project includes a Makefile to simplify the setup and execution process. 

### 1. Start the Infrastructure
Bring up all services (PostgreSQL, MinIO, API, Worker, Prometheus, and Grafana) using Docker Compose:
```bash
make up
```

### 2. Run Database Migrations
Initialize the database schema:
```bash
make migrate
```

### 3. Seed the Database
Populate the database with demo data (2 companies, 6 users, and 1 partner with an API key):
```bash
make seed
```

After seeding, the following users are available (password is `password` for all):
- **Company Admins**: `admin@acme.com`, `admin@beta.com`
- **Employees**: `employee@acme.com`, `employee@beta.com`
- **Provider**: `provider@demo.com`
- **Platform Admin**: `padmin@provenn.io`

### 4. Run the Automated Demo
You can run the end-to-end interview demo script, which simulates the complete story (issuance, download, match, approval, tamper, mismatch, rejection, and export):
```bash
make demo
```

## Running the Web Frontend

The Next.js frontend connects to the local API.

1. Navigate to the `web` directory:
```bash
cd web
```
2. Install dependencies (if you haven't already):
```bash
npm install
```
3. Start the development server:
```bash
npm run dev
```

You can now access the frontend at `http://localhost:3000`.

- **Login**: `http://localhost:3000/login`
- **Employee Portal**: `http://localhost:3000/` (Submit invoices for verification)
- **Company Admin Portal**: `http://localhost:3000/admin` (Approve/reject verifications and export)
- **Platform Admin Console**: `http://localhost:3000/console` (Cross-tenant analytics)

## Observability

The system is fully instrumented with Prometheus.

- **Grafana Dashboard**: Available at `http://localhost:3001` (Credentials: `admin` / `admin`).
- Navigate to the provisioned **"ProveNN API"** dashboard to view request rates, latencies, worker job statuses, and business metrics (e.g., invoices issued).

## Architecture

- **`cmd/api`**: The main HTTP server (Chi router).
- **`cmd/worker`**: The background job processor (River queue) for hashing and stamping PDFs.
- **`internal/`**: Core domain logic, separated into packages (`auth`, `invoice`, `verification`, `billing`, `admin`, `db`, `observability`, `storage`).
- **`web/`**: The Next.js React frontend.
- **`deploy/`**: Docker Compose and configuration files.

## Testing

```bash
PROVENN_E2E=1 go test ./internal/e2e/... -v -count=1
```

To run standard unit tests:
```bash
go test ./...
```
